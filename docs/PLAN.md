# Elegant Clock 开发规划

## 产品目标

Elegant Clock 是面向 Windows 与 Linux 的桌面时钟软件，重点是轻量、美观、可透明显示、可个性化，并提供日常计时工具。

## 核心功能

- 当前时间：大字号显示系统本地时间，包含秒。
- 日期显示：显示星期、年月日。
- 透明显示：用户可切换透明/不透明窗口效果，并调整透明度。
- 显示个性化：支持字体、时钟字号、字体颜色和背景颜色设置。
- 默认视图：启动后只显示时钟，设置和功能分别由右下角齿轮/三横杠打开独立窗口。
- 倒计时：支持按小时/分钟/秒设置，也支持倒计时到指定日期时间。
- 正计时：提供开始、暂停、重置的秒表功能。
- 番茄钟：支持专注、短休息、长休息和长休间隔配置。
- 提醒：支持类似日历的指定日期时间提醒，包含标题、备注、本地保存和系统通知。
- 铃声：内置默认铃声 `ringtone_default.mp3`，也支持用户选择本地音频文件，并支持试听/停止试听。
- 开机自启动：安装时询问是否配置，安装后可在设置中再次调整。
- 窗口体验：无边框窗口、最小化、关闭、最大化/还原、可选置顶、托盘后台运行。
- 闲置紧凑模式：超过 5 秒无操作后仅显示圆角时钟和日期；点击恢复完整窗口，拖动只移动位置。

## 技术路线

- 桌面框架：Electron，覆盖 Windows 与主流 Linux 桌面环境。
- 前端实现：原生 HTML/CSS/JavaScript，降低依赖复杂度。
- 本地状态：主进程保存透明显示、置顶偏好、外观配置、铃声选择、番茄钟、倒计时、正计时和提醒列表，确保服务启动后计时任务继续运行。
- 桌面提醒：通过 Electron `Notification` API 发送系统通知，并在主时钟窗口播放用户选择的铃声。
- 开机自启动：Windows 使用当前用户登录启动项；Linux 运行时使用 XDG Autostart，适配 GNOME、KDE、Cinnamon、LXQt、Xfce 等主流桌面。
- 安装阶段自启动：Windows NSIS 安装器提供图形化选择；Linux deb/rpm 在交互式终端安装时询问，非交互安装时跳过并允许用户在设置中启用；Arch 包安装后提示用户在设置中启用或手动复制 desktop 文件。
- 单实例与后台：重复启动时激活已有窗口，右上角可隐藏到状态栏，点击状态栏图标恢复完整窗口。
- 独立窗口：设置窗口和功能窗口不参与 5 秒闲置紧凑模式；功能窗口选择项目后在原窗口切换到主界面，并提供返回键。
- 关于窗口：设置底部提供独立关于窗口，显示作者 GitHub 主页、项目主页与当前仓库许可证。
- 安全边界：开启 `contextIsolation`，通过 `preload` 暴露最小 IPC API。
- 构建工具：`electron-builder` 生成 Windows 与 Linux 安装包。

## 打包交付

- Windows：`.exe` NSIS 安装器、`.msi` 安装器。
- Linux：`.deb` Debian 安装包、`.dpkg` Debian 包别名、`.rpm` RPM 安装包、`.pkg.tar.zst` Arch Linux pacman 安装包。
- 校验文件：Release Assets 同时包含 `SHA256SUMS.txt`，整合列出每个安装包的 SHA256。

## 里程碑

1. v0.1.0：实现基础时钟、日期、透明切换、倒计时、正计时与 Release 打包。
2. v0.2.0：增加番茄钟、日历式提醒、系统通知和 Release SHA256 校验文件。
3. v0.3.0：增加默认纯时钟视图、透明度、字体、字号、字体颜色、背景颜色和可选铃声。
4. v1.0.0：增加闲置紧凑视图、托盘后台运行、单实例激活、关于窗口、开机自启动设置与安装时自启动提示，形成首个稳定大版本。
5. v1.0.1：设置与功能改为独立窗口，增加铃声试听停止按钮，倒计时状态同步到主时钟，计时和提醒迁移到主进程持久状态服务。

## 风险与限制

- Linux 透明窗口效果依赖桌面环境与合成器，部分发行版可能显示为半透明或不透明。
- Windows 安装包未配置代码签名时，系统可能提示未知发布者。
- `.dpkg` 不是 Debian 官方常用扩展名，本项目会额外复制一份 `.deb` 内容作为 `.dpkg` 资产以满足发布资产命名需求。
- Arch Linux 用户可使用 `sudo pacman -U Elegant-Clock-*-Arch-x86_64.pkg.tar.zst` 安装 Release 中的 pacman 包。
